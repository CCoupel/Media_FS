# Skill : Déploiement du site marketing GitHub Pages

Ton rôle est de générer ou mettre à jour le site marketing de Media_FS sur la branche `gh-pages` du repo `CCoupel/Media_FS`.

## Ce que tu dois faire

### 1. Collecter les données du projet

Exécute ces commandes pour récupérer l'état actuel :

```bash
# Version actuelle (dernier tag git ou go.mod)
git describe --tags --abbrev=0 2>/dev/null || echo "v0.1.0-dev"

# Issues ouvertes par label
gh issue list --state open --json number,title,labels,createdAt --limit 50

# Milestones et leur progression
gh api repos/CCoupel/Media_FS/milestones --jq '[.[] | {title, open_issues, closed_issues, state}]'

# Dernier commit
git log -1 --format="%h|%s|%ai"

# Nombre de releases
gh release list --limit 10 2>/dev/null || echo "Aucune release"

# Dernière release
gh release view 2>/dev/null || echo "pre-release"
```

Lis également :
- `SPECS.md` — pour les fonctionnalités et la roadmap P0→P3
- `go.mod` — pour la version Go et le module path
- `docs/ARCHITECTURE.md` — pour le diagramme d'architecture

### 2. Générer le site

Génère un fichier `index.html` complet. Le site doit :

**Design**
- Thème sombre, couleur accent `#3b82f6` (bleu), police système
- Totalement responsive (mobile + desktop)
- Pas de dépendance externe (no CDN, no frameworks) — tout inline

**Sections obligatoires (dans cet ordre)**

1. **Hero** : nom du projet, tagline, badge version live (JS), bouton "View on GitHub" → `https://github.com/CCoupel/Media_FS`

2. **Stats bar** : compteurs live en JS depuis GitHub API
   - Nombre d'issues ouvertes
   - Dernière mise à jour (date du dernier commit)
   - Langage principal : Go

3. **Features** : 6 cartes reprenant les fonctionnalités clés issues de SPECS.md
   - VFS Windows (WinFSP) + Linux (FUSE)
   - Multi-serveurs Jellyfin / Emby / Plex
   - Browse + streaming direct (VLC, MPV)
   - Copie locale multi-threads (HTTP Range)
   - Métadonnées NFO + artwork Kodi-compatible
   - Interface de configuration (system tray + UI web locale)

4. **Architecture** : diagramme ASCII dans un bloc `<pre>` stylisé, repris de `docs/ARCHITECTURE.md`

5. **Roadmap** : tableau P0→P3 avec statut visuel (✅ done / 🔄 in progress / 📋 planned) basé sur les milestones GitHub (ou SPECS.md si pas de milestones)

6. **Issues ouvertes** : liste live générée en JS depuis `https://api.github.com/repos/CCoupel/Media_FS/issues?state=open&per_page=10`, avec titre, labels colorés, lien direct

7. **Install / Getting started** : bloc de code avec les prérequis et la commande de build

8. **Footer** : lien GitHub, licence MIT, "Powered by Go + WinFSP/FUSE"

**JavaScript embarqué** — au chargement de la page, le JS doit :
```javascript
// Appels GitHub API (pas d'auth nécessaire, repo public)
const API = 'https://api.github.com/repos/CCoupel/Media_FS'
// Fetch: /releases/latest → version badge
// Fetch: /issues?state=open → compteur + liste
// Fetch: /commits?per_page=1 → date dernier commit
// Gestion d'erreur silencieuse si rate limit atteint
```

### 3. Déployer sur gh-pages

```bash
# Créer un worktree isolé sur la branche gh-pages
git worktree add /tmp/mediafs-pages gh-pages 2>/dev/null || (
  git worktree add /tmp/mediafs-pages --orphan gh-pages 2>/dev/null
)

# Copier le fichier généré
cp index.html /tmp/mediafs-pages/index.html

# Commit et push
cd /tmp/mediafs-pages
git add index.html
git commit -m "chore(pages): update marketing site

Auto-generated by /gh-pages skill.
Syncs: version, open issues, roadmap from SPECS.md

Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>"
git push origin gh-pages --force

# Nettoyage
cd -
git worktree remove /tmp/mediafs-pages --force
```

### 4. Activer GitHub Pages (si pas encore fait)

Vérifie avec :
```bash
gh api repos/CCoupel/Media_FS/pages 2>/dev/null || echo "Pages not enabled"
```

Si pas activé :
```bash
gh api repos/CCoupel/Media_FS/pages \
  --method POST \
  -f source[branch]=gh-pages \
  -f source[path]=/
```

### 5. Confirmer

Affiche à l'utilisateur :
- L'URL du site : `https://ccoupel.github.io/Media_FS/`
- Le résumé des données incluses (version, nb issues, statut roadmap)
- Un rappel : relancer `/gh-pages` à chaque release ou évolution majeure

## Notes importantes

- Le site est **statique** mais avec données **live** via JS + GitHub API publique (60 req/h sans token, suffisant)
- Ne jamais toucher à la branche `main` depuis ce skill
- Si la branche `gh-pages` n'existe pas, la créer en orphan (sans historique de `main`)
- Le fichier `index.html` est le seul fichier nécessaire (pas de Jekyll, pas de framework)
