Everything you need to browse your library natively

No extra client needed — your OS file manager becomes your media browser.

🪟
Windows Explorer & Linux file managers
Powered by WinFSP on Windows and FUSE 3 on Linux. Your library appears as a real drive letter (e.g. Z:) — no special client required.
WinFSPFUSE 3
🔗
Multi-server, unified tree
Connect Jellyfin, Emby and Plex servers simultaneously. Each appears as a user@server folder with its own library icon.
JellyfinEmbyPlex P3
▶️
Stream directly from any app
Open a .mkv in VLC, MPV or any player straight from Explorer. HTTP Range requests enable instant seek without full download.
Read-only · safe
⚡
Parallel chunk download
Copying a file locally? Media_FS splits it into N chunks downloaded simultaneously via HTTP Range — configurable thread count and chunk size.
4 threads default
📄
NFO + artwork sidecars
Every media item ships with a Kodi-compatible .nfo and artwork files (poster.jpg, fanart.jpg) — generated on the fly from the server API.
Kodi-compatible
🔧
System tray + web config UI
Media_FS lives in the system tray. Click "Configuration" to open a local web UI to add/remove servers, test connections and adjust settings.
No install wizard

Get Started

CGO is required — WinFSP or libfuse3 links at compile time.

1

Install prerequisites

Windows: install WinFSP + GCC via MinGW  |  Linux: sudo apt install libfuse3-dev gcc

2

Clone and build

# Clone
git clone https://github.com/CCoupel/Media_FS.git && cd Media_FS

# Build (Windows)
CGO_ENABLED=1 GOOS=windows go build -o dist/mediafs.exe ./cmd/mediafs

# Build (Linux)
CGO_ENABLED=1 go build -o dist/mediafs ./cmd/mediafs
3

Configure a server

Launch Media_FS — the tray icon appears. Click Configuration… to open the local web UI and add your Jellyfin or Emby server (URL + API key).

4

Mount and browse

Click Mount All in the tray. Your library appears on drive Z: (Windows) or /mnt/mediafs (Linux). Open a film directly in VLC.
